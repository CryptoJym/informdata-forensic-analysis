Verification Triple Field Pattern

Employment Verification Response Pattern

/Downloads/Responses/Verification/Employment Verification Response.json 
"employmentVerification": {
    // Pattern 1: Employer field
    "employer": "Sendoso",                 // Base field
    "employerVerified": true,             // Verification status
    "employerDiscovered": "Sendoso",      // What was found
    
    // Pattern 2: Title field
    "title": "Send Curator",               // Base field
    "titleVerified": true,                // Verification status
    "titleDiscovered": "Send Curator",    // What was found
    
    // Pattern 3: Employment dates
    "employedFrom": "2021-11-01",          // Base field
    "employedFromVerified": true,        // Verification status
    "employedFromDiscovered": "11/29/2021", // What was found (different format!)
    
    "employedTo": "2022-10-01",            // Base field
    "employedToVerified": false,         // ❌ Not verified
    "employedToDiscovered": "Present",   // Different value found!
}
Critical Pattern: EVERY field in a verification response has three versions. Missing any of these means data loss.

Verification Pattern Rules

Base Field
Contains the provided value
fieldVerified
Boolean - was it verified?
fieldDiscovered
String - actual value found

Missing Core Features - API Requirements

JWT Authentication Requirement

Every API Endpoint

❌ Current Implementation

// No authentication headers found
const response = await fetch(url, {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json'
        // Missing Authorization header!
    },
    body: JSON.stringify(data)
});

✅ Required Implementation

// From API Documentation
const response = await fetch(url, {
    method: 'POST',
    headers: {
        'Authorization': `Bearer ${jwtToken}`,
        'Content-Type': 'application/json'
    },
    body: JSON.stringify(data)
});
